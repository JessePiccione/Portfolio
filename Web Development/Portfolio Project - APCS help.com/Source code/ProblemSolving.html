Solving a Problem

The first step to solving any problem is to translate the words of the problem. For example:
Create a method that takes in two integers. The program will return the sum of the ints times the smaller integer:
This is pretty vague instuction wise all this says is what to take in and return. Lets think:


//we need to write a method takes in two ints as parameters
//get the sum of the ints
//find the smaller int
//return the sum * smaller int

So is that easier to read? Is it more clear on how to solve the problem? We then can take our words and translate that into Java code:


//we need to write a method takes in two ints as parameters 
                
public int sumTimesSmaller(int one, int two){

    //get the sum of the ints
                        
    int sum = one + two;

    //find the smaller int
                        
    int smaller = one;
                    
    if(smaller > two){
                    
        smaller = two;
                    
    }
                        
    //return the sum * smaller int

    return sum * smaller;

}

This example is easy but demostrates the thought process to build a solution. The process can be broken down into a few steps.

My Steps to Solving a Problem

  • Get that AP CS Appendix Ready
  • Step One - read the problem.
  • Step Two - translate the problem into your own words.
  • Step Three - identify what tasks you need to perform.
  • Step Four - code your tasks for the problem.
  • Step Five - debug your code for logical and syntax errors.
