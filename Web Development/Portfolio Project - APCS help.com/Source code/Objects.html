What's an Object?

There are two essential types of data that allow Java to work. Primitive data types like int, char, double, float, long, and boolean. Objects are essentially containers of data. With the data stored, several methods can be stored in an Object's class to perform tasks for the Object's needs. All objects are children of a class known as Object. Object contains methods which can be overidden by its children's class. Common methods like toString() and equals(Object obj) are inherited from Object. However, the classes can specify there own methods as long as the method headers just need to match to properly overide. With that being said here is a basic class for x and y points:

                
public class Point implements Comparable{

    //instance data

    private int x;
                    
    private int y;

    //overloaded constructers
                       
    //default 
                       
    public Point(){
                        
        this.x = 0;
                        
        this.y = 0;
                        
    }
                       
    //parameterized contructer
                       
    public Point(int x, int y){
                        
        this.x = x;
                        
        this.y = y;
                        
    }

    //accessor methods "getters"

    public int getX(){

        return x;

    }

    public int getY(){

        return y;

    }

    //mutator methods "setters"

    public void setX(int x){

        this.x = x;

    }

    public void setY(int y){

        this.y = y;

    }

    //Comparable methods 

    public int compareTo(Object obj){
                            
        Point p = (Point)obj;
                            
        double size1 =(Math.sqrt((this.x * this.x)+(this.y*this.y)));
                            
        double size2 = (Math.sprt((p.getX() * p.getX()) +(p.getY() * p.getY())));
                            
        if(size1 < size2){
                            
            return -1;
                            
        }
        else if(Math.abs(size1-size2) <= 0.001){
                            
            return 0;
                            
        }

        return 1;

    }

    // equals overide

    public boolean equals(Object obj){

        Point p = (Point)obj;

        return this.x == p.getX() && this.y == p.getY();

    }

    //toString overide 

    public String toString(){

        return "(" + x + ", " + y + ")";
                        
    }

}

This may seem like a lot right?

The Structure of the Class

  • Instance Data - data specified for each object that is created
  • Defualt Constructer - a constructer that takes in no data to instantiate the object
  • Parameterized Constructer - a constructer which takes in data to instantiate the object
  • Accessor or getter methods - methods that return the data of the object
  • Mutator or setter methods - methods that change the data of the object
  • Comparable interface - requires compareTo method in the class
  • equals method overide - overides equals method from parent class Object
  • toString method overide - overides the toString method from parent class Object
