What is Recursion?

To understand recursion, you must first understand recursion. Recursion, in Java and other languages, is the action of a method calling itself. Acursion, recursion's brother, is when a method calls another method. Working with recursion is simaler to loops in terms of operation. In fact most recursive methods can be written with a loop instead.

Take a look at this method: 

public static int factorial(int n){

    if(n <= 1){

        return 1;
                    
    }
                    
    return n * factorial(n-1);

}

This is method returns the factorial of n. If n was 5, the execution of the method looks like:

factorial(5); which returns 5 * factorial(4)
factorial(4); which returns 4 * factorial(3)
factorial(3); which returns 3 * factorial(2)
factorial(2); which returns 2 * factorial(1)
factorial(1); which returns 1

So factorial(5) really returns:

5 * 4 * 3 * 2 * 1 OR 120

We can write this with a for loop to complete the same task:


public static int factorial(int n){
                   
    int buffer = 1;
                   
    for(int x = n; x > 0; x--){

        buffer *= x;

    }

    return buffer;

}

So, whats the point of using recursion? If we can just loop do the same thing with a loop why use it? Well, recursive methods are not hard on the eyes. They usaully require less code than a for loop. Notice the recursive method has around three lines. If you want to picky the recursive method can be written in one line like This:


public static int factorial(int n){

    return (n <= 1)? 1 : (n*factorial(n-1));
                    
}

That is called a ternary if or inline if statement. That will be covered in some solution videos in the future. Refrain from using that if you do not understand it, because it is not as high level as the rest of Java. Recursion is cool but more of an advanced programming concept. On the AP CS Test, you will not need to write a recursive method. However, you still might run into a multiple choice problem or two.

Recursive Methods: Things to Remember

  • Terminating Condition - a condition that will terminate the recursive method when hit like when n <= 1.
  • Changing Data - something needs to be changed before making a recusive call like n-1.
  • Recursive call - a method is not recursive without calling itself.
  • Infintite Recursion - a recursive method that will never terminate, usaully due to a logical error.
